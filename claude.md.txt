# ERP Asset Management System

## Project Overview
This is a production-style ERP Asset Management System built using:
- React.js frontend
- Node.js + Express backend
- MySQL database

The goal is to evolve this into an enterprise-grade ERP platform with scalable architecture, analytics, ticket management, audit tracking, notifications, and role-based access control.

---

# Folder Structure

## Frontend
Location: /Frontend

Main responsibilities:
- Dashboard UI

## Backend
Location: /Backend

Architecture:
- routes/

- middleware/

- utils/

Follow MVC + service-layer architecture.

---

# Coding Rules

- Do NOT rewrite the entire project
- Extend existing architecture incrementally
- Reuse existing components and APIs
- Keep endpoints stateless
- Use async/await consistently
- Keep business logic in services
- Avoid duplicate code
- Follow existing naming conventions
- Do not modify .env or deployment configs
- Use clean and maintainable code

---

# ERP Challenges To Solve

This ERP should solve real-world operational problems:

1. Slow dashboard performance
2. Weak ticket management
3. No audit tracking
4. Poor asset lifecycle tracking
5. Missing notifications
6. Lack of analytics
7. Weak role-based security
8. Duplicate/inconsistent data
9. Poor mobile responsiveness
10. Scalability problems

Before coding:
1. Analyze existing structure
2. Explain implementation plan
3. Wait for approval
4. Implement phase by phase

After each phase:
- summarize changed files
- explain architecture decisions
- pause for review

# Constraints

- adding dependencies
- deleting files
- major refactors
